SNAP Data Processors - Idepix MODIS Algorithm Specification

Algorithm Specification

The Idepix classification algorithm for MODIS is based on a neural network approach. The neural net used was trained to finally provide per pixel one of the properties CLEAR, CLOUD_AMBIGUOUS, CLOUD_SURE, SNOW_ICE. The following MODIS bands are used as input for the neural net:

  • EV_250_Aggr1km_RefSB.1
  • EV_500_Aggr1km_RefSB.3
  • EV_500_Aggr1km_RefSB.4
  • EV_500_Aggr1km_RefSB.5
  • EV_500_Aggr1km_RefSB.7
  • EV_1KM_Emissive.23
  • EV_1KM_Emissive.25
  • sqrt_EV_1KM_RefSB.26
  • EV_250_Aggr1km_RefSB.1
  • EV_250_Aggr1km_RefSB.1
  • EV_250_Aggr1km_RefSB.1

In a second step, the neural network classification is refined using two additional quantities, the 'whiteness' and the 'brightness', based on the approach described in more detail in [1]. E., Baum, B.,Wetzel-Seeman, S., and Zhang, H., 2002.))

where E0 and θ are the solar spectral irradiance and the sun zenith angle, respectively.

The conversion from reflectances to radiances simply follows the inverse of this equation.

For all sensors supported by the Radiance-to-Reflectance Processor, the sun zenith angle is provided per pixel with the L1b products. The solar spectral irradiance values are also provided per pixel in case of OLCI L1b products, whereas for SLSTR MERIS they are taken from auxiliary data.
